# SKILL 设计指南：避免大模型歧义

## 问题诊断

如果你的 SKILL 被调用时出现以下问题，说明 SKILL 设计需要改进：
- 大模型重新编写了已有功能
- 大模型偏离了 SKILL 定义的规则（如命名规则）
- 完成任务后发现需要多次提醒大模型查看 SKILL
- 花费大量 token 和时间重复造轮子

## 设计原则

### 1. 规则层级体系

必须明确区分三类规则：

```
🔴 禁止行为 - 大模型绝对不能做的事
   ├─> 不要重新编写已有功能
   ├─> 不要偏离核心规则
   └─> 不要跳过 SKILL 函数直接操作

✅ 强制规则 - 大模型必须遵守的规则
   ├─> 标准工作流调用方式
   ├─> 核心函数使用要求
   └─> 配置文件读取规则

🔵 灵活区域 - 大模型可以自由发挥的场景
   ├─> 数据筛选和过滤
   ├─> 定制化导出
   └─> 组合多个函数
```

### 2. 判断流程图

大模型需要一个清晰的决策树：

```
用户请求
    │
    ├─> 标准任务？
    │   ├─> 是 → 使用标准工作流函数
    │   └─> 否 → 继续判断
    │
    ├─> 需要核心功能？
    │   ├─> 是 → 必须使用 SKILL 提供的函数
    │   └─> 否 → 继续判断
    │
    ├─> 需要定制处理？
    │   ├─> 是 → 先用 SKILL 函数 → 再自由处理数据
    │   └─> 否 → 继续判断
    │
    └─> 其他需求？
        └─> 检查 SKILL 是否有对应函数
```

### 3. 核心函数速查表

创建一个表格，让大模型快速找到应该使用的函数：

| 需求 | 必须使用的函数 | 位置 |
|------|----------------|------|
| XXX 功能 | `xxx_function()` | core/xxx.py |

### 4. 使用示例分类

提供三类示例：

**标准场景** - 直接调用 SKILL 函数
```python
from workflow import standard_workflow
standard_workflow("input/")
```

**定制场景** - 先 SKILL 函数，再自由处理
```python
# 1. 使用 SKILL 函数（必须）
data = parse_data("input/")

# 2. 自由筛选（灵活）
filtered = [d for d in data if condition]

# 3. 使用 SKILL 函数（必须）
export_data(filtered, "output/")
```

**混合场景** - 组合多个 SKILL 函数
```python
func1("input/")
func2("output/")
```

```markdown
---
name: your-skill-name
description: |
  简短描述触发场景
  明确列出用户会说什么来触发此 SKILL
---

# SKILL 名称

## 大模型调用规则（强制）

**以下规则是大模型调用此 SKILL 时必须遵守的约束**

### 🔴 禁止行为

1. **禁止重新编写核心功能代码**
   - 不要自己写 XXX 代码 → 使用 `XXXFunction`
   - 不要自己写 YYY 代码 → 使用 `YYYFunction`

2. **禁止偏离核心规则**
   - 必须使用：XXX 格式
   - 不可自定义（用户明确要求除外）

### ✅ 必须遵守的规则

1. **标准工作流调用**
   ```python
   from workflow import standard_workflow
   standard_workflow("input/")
   ```

2. **核心功能必须使用**
   ```python
   from core.module import CoreFunction
   func = CoreFunction()
   result = func.process("input")
   ```

3. **配置/规则要求**
   - 配置文件：自动加载，不要硬编码
   - 命名规则：必须遵守 XXX 格式

### 🔵 灵活调整区域

**以下场景大模型可以自由发挥，但必须基于 SKILL 的核心函数**

1. **数据筛选和过滤**
2. **定制化导出**
3. **组合多个函数**

```
用户请求 → 判断 → 选择正确的调用方式
```

## 验证检查清单

在发布 SKILL 前，检查以下问题：

### metadata 检查
- [ ] `name` 简洁且能反映功能
- [ ] `description` 明确列出触发场景
- [ ] `description` 使用第三人称（"This skill should be used when..."）

### SKILL.md 检查
- [ ] 有明确的 🔴 禁止行为 部分
- [ ] 有明确的 ✅ 强制规则 部分
- [ ] 有明确的 🔵 灵活区域 部分
- [ ] 有判断流程图
- [ ] 有核心函数速查表
- [ ] 有三类使用示例

### 结构检查
- [ ] SKILL.md < 200 行（保持精简）
- [ ] 详细技术文档在 references/
- [ ] 可复用脚本在 scripts/
- [ ] 模板文件在 assets/

## 常见错误

| 错误 | 后果 | 解决方案 |
|------|------|----------|
| 只有功能说明，没有规则层级 | 大模型会自己重写代码 | 添加 🔴 禁止行为 |
| 没有判断流程图 | 大模型不知道选哪个函数 | 添加决策树 |
| 没有使用示例 | 大模型调用方式不一致 | 添加三类场景示例 |
| SKILL.md 太长 | 大模型找不到关键信息 | 移动详细内容到 references/ |
| 没有函数速查表 | 大模型找不到对应函数 | 添加速查表 |

## 持续改进

使用 SKILL 后，如果发现大模型仍然偏离：

1. **检查日志** - 看大模型在哪个步骤出问题
2. **加强规则** - 在对应位置添加更明确的禁止/强制规则
3. **添加示例** - 针对问题场景添加具体示例
4. **简化流程** - 如果判断流程太复杂，考虑简化
