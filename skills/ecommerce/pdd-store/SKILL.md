# 拼多多个人店全自动开店 Skill

## 触发条件

用户提到以下关键词时触发：
- 拼多多、PDD、开店、电商、选品、货源、代发

## 核心理念

Hermes 可完成 90% 的开店工作，用户只需：
1. 注册店铺（身份证实名）
2. 1688 付款（涉及资金）

## 关键工具：OpenCLI

**1688反爬很强，Camofox也被拦。必须用OpenCLI：**

```bash
# 安装
npm install -g @jackwener/opencli

# 搜索商品
opencli 1688 search "关键词" --limit 5

# 获取商品详情（含图片URL）
opencli 1688 item <offer_id>

# 下载商品图片
opencli 1688 download <offer_id> --output ./images

# 查看店铺信息
opencli 1688 store <shop_id>
```

**前提：Chrome必须登录1688**（OpenCLI复用浏览器登录态）

## 完整工作流（9步）

### Step 1: 选品调研
```bash
# 用 anysearch 搜拼多多热销数据
# 关键词：拼多多 热销榜 家居日用 爆款 月销量
# 输出：爆款清单 + 利润预估
```
**⚠️ 避免撞品策略：**
- 不要只搜大品类（硅胶厨具、抹布），要搜细分品类（麦秸秆厨具、鱼鳞抹布）
- 搜拼多多热搜词找蓝海：`anysearch 搜索 "拼多多 搜索热词 蓝海"`
- 看竞品差评找痛点：差评多说明有改进空间
- 跨平台选品：抖音/小红书什么火→搬到拼多多
- 搜冷门但刚需的品：如中药泡煮杯、宠物慢食碗

**注意事项：**
- 个人店只能卖家居日用类（服饰、家居、日用、宠物、饰品）
- 不能卖食品、药品、美妆、3C数码（需要资质）
- 优先选：高复购、轻便、无售后的产品

### Step 2: 找货源

**⚠️ 必须用 OpenCLI 搜1688，不要用浏览器自动化（会被反爬拦截）**

```bash
# 安装 OpenCLI（一次性）
npm install -g @jackwener/opencli

# 搜索商品（复用 Chrome 登录态，不消耗 token）
opencli 1688 search "硅胶厨具套装 木柄" --limit 3

# 获取商品详情（价格、规格、卖家信息）
opencli 1688 item <offer_id>

# 下载商品图片
opencli 1688 download <offer_id> --output ./images
```

**OpenCLI 1688 命令：**

| 命令 | 功能 |
|------|------|
| `opencli 1688 search "关键词"` | 搜索商品 |
| `opencli 1688 item <id>` | 商品详情 |
| `opencli 1688 download <id>` | 下载图片/视频 |
| `opencli 1688 store <shop_id>` | 店铺信息 |
| `opencli 1688 assets <id>` | 列出可下载素材 |

**选供应商标准：**
- ✅ "实力商家"/"源头工厂" 标识
- ✅ 开店 3 年以上
- ✅ 回头率 20%+
- ⚠️ 一定要先买样品测试

**利润计算公式：**
```
利润 = 拼多多售价 - 1688拿货价 - 运费(5-8元) - 平台费(0.6%)
目标：单品利润 > 10元
```

**⚠️ 链接必须验证：** 搜索结果拿到链接后，用 `opencli 1688 item` 确认商品内容跟我们要卖的一致，不能直接用搜索结果里的链接。

## 经验教训

- **OpenCLI 是1688搜货的唯一可靠方式** — 浏览器自动化（包括 Camofox）会被1688反爬拦截，OpenCLI 复用 Chrome 登录态可以绕过
- **1688链接必须验证** — 搜索结果拿到链接后，用 `opencli 1688 item <id>` 确认商品内容，不能直接用搜索结果里的链接
- **用户纠正了工作流思路** — 应该先搜数据→拿1688价格→算利润，而不是反过来
- **用户指出后台分析、确认发货、客服回复都可以用浏览器自动化完成**
- **成本分析要实事求是** — 不要高估日均token消耗（选品和生图是一次性的，日常只有客服和数据分析）
- **店铺名禁用词** — 物美、优质、精品、名牌、官方、旗舰等不能用，"优选"可以用
- **个人店可能不需要保证金** — 2026年6月发布商品时没有提示
- **不要纠结产品图片** — 用户多次纠正我不要花太多时间在图片上，重点是选品和货源
- **文字转图片比AI生图更快** — 用户要的是白底黑字的文字图片，不是AI生成的产品图。用PIL直接把文案做成图片，比调Gemini API更稳定快速
- **详情页用图片比纯文字好** — 批量上传图片比手动打字快，用户可以直接看图复制文字
- **OpenCLI可能限流** — 1688搜索/详情接口偶尔会返回空结果，重试或换关键词即可

## 文字转图片模板（详情页用）

```python
from PIL import Image, ImageDraw, ImageFont

def create_text_image(title, lines, filename, font_path='C:/Windows/Fonts/msyh.ttc'):
    """生成白底黑字的文案图片，用于拼多多详情页"""
    width = 750
    padding = 30
    line_height = 36
    
    title_font = ImageFont.truetype(font_path, 30)
    content_font = ImageFont.truetype(font_path, 20)
    
    # 精确计算高度（防止截断）
    height = padding + 50 + 20 + len(lines) * line_height + 60
    
    img = Image.new('RGB', (width, height), 'white')
    draw = ImageDraw.Draw(img)
    
    y = padding
    draw.text((padding, y), title, fill='black', font=title_font)
    y += 50
    draw.line([(padding, y), (width - padding, y)], fill='gray', width=2)
    y += 20
    
    for line in lines:
        draw.text((padding, y), line, fill='black', font=content_font)
        y += line_height
    
    img.save(filename, quality=95)
```

**使用场景：**
- 把上架文案.md的文字做成图片
- 用户看图复制文字粘贴到拼多多
- 比手动打字快，比AI生图稳定

## 经验教训

### 1688相关
- **搜索结果的链接不能直接用** — 必须用OpenCLI的`1688 item`验证商品内容，否则链接和产品不对应
- **OpenCLI是1688的神器** — 复用Chrome登录态，能绕过反爬，获取商品详情、价格、图片
- **每个SKU要对应不同图片** — 3件套/8件套/12件套的预览图不能用同一张
- **备2-3家供应商** — 防止一家断货没法发货
- **先买样品验证质量** — 不要直接大批量下单

### 上架相关
- **品牌选"无品牌"** — 个人店家居日用品不需要品牌授权
- **24小时发货有风险** — 1688供应商不一定能24小时发，先选48小时
- **SKU默认选中利润款** — 不要默认选引流款，容易亏本
- **原价设拼单价的1.5-2倍** — 显示有折扣，吸引买家拼团
- **库存初期设10件** — 避免超卖，卖完再补

### 图片相关
- **Gemini生图会被限流** — 遇到503错误等几分钟再试
- **文字转图片用PIL更稳定** — 不依赖API，白底黑字清晰好读
- **图片高度要精确计算** — 不然文字会被截断，用 `行数×行高+上下padding` 计算
- **文件名不能有特殊字符** — Windows不支持 `/:*?"<>|`，需要sanitize
- **SKU图必须找全** — 每个规格（3件套/8件套/12件套）都要有对应图片，不能只找一张通用图
- **1688商品图可能跟实际不符** — 下载后要用analyze_image验证图片内容是否正确
- **详情页图片要对应产品** — 硅胶厨具的文案图不能用到手机防水袋上

### 文案相关
- **产品内容不能搞混** — 硅胶厨具的文案不能用到手机防水袋上
- **标题要包含热搜关键词** — 用 anysearch 搜拼多多热词
- **详情页用图片比文字好** — 买家更愿意看图，不想看大段文字

### 定价相关
- **引流价吸引点击，利润价赚钱** — 不要把所有价格都设很低
- **套餐价提高客单价** — 2只装/3只装比单只更划算
- **利润率目标50%+** — 1688拿货价通常很低，定价不要太保守
- **定价文档要写清楚每个SKU** — 规格名称、拼单价、原价、库存、对应图片，不能模糊
- **拼单价和原价要同时填** — 拼单价是低价，原价是高价，显示有折扣
