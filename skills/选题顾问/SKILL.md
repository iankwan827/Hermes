## 核心概念（2026年1月更新）

> 以下是对选题、标题、开头关系的明确定义。

### 选题 vs 内容

**选题：** 你要讲什么事情（what）
**内容：** 如何讲这件事情（how）

### A类和B类选题

**A类选题（稀缺）：**
- 天然吸引力强（我和马云吃了个饭）
- 特点：话题=选题=标题=开头
- 策略：直接说就行，不需要包装

**B类选题（常见）：**
- 天然吸引力弱（上班对创业的影响）
- 特点：需要通过标题和开头来包装
- 策略：标题包装选题，开头快速验证价值

### 选题评估的核心

**判断是A类还是B类：**
- A类：直接推荐
- B类：评估能否包装成看起来像A类

**如果是B类选题：**
- 标题的任务：扩大话题、承诺价值
- 开头的任务：快速建立话题、快速展示价值、快速建立可信度

| 优势 | 说明 | 证据 |

### Step 1: 话题范围检查

问三个问题：
1. 不做自媒体的人会看吗？
2. 做实体生意的人会看吗？
3. 上班族会看吗？

**判断：**
- 3个都是「是」→ 广选题，流量潜力大
- 2个「是」→ 中等选题
- 3个都是「否」→ 窄选题，流量上限低，建议调整

### Step 2: 优势匹配检查

这个选题能发挥哪个优势？
- [ ] 数据背书（有具体数字吗？）
- [ ] 商业洞察（有独特视角吗？）
- [ ] 认知冲突（反常规吗？）
- [ ] 平台信息差（有独家信息吗？）
- [ ] 故事/戏剧性（有冲突反转吗？）

**判断：**
- 勾选 2 个以上 → 有竞争力
- 勾选 1 个 → 竞争力一般
- 一个都没有 → 选题没有差异化，建议调整

### Step 3: 劣势规避检查

这个选题是否踩中劣势？
- [ ] AI工具教程（操作演示类）
- [ ] 知识付费方法论（只有从业者看）
- [ ] 纯观点无冲突（温和陈述）
- [ ] 行业分析门槛高（普通人看不懂）

**判断：**
- 勾选任意一项 → 高风险，建议调整或放弃

### Step 4: 致命伤检查（v3.0 新增）

即使前面几步都通过，以下任何一条成立就直接否定选题，不进入改造：

**4.1 视角归属检查：观众觉得你在说谁？**
- 听到这个选题的第一反应是"对，那些人确实是这样"（说别人）还是"卧槽说的是我"（说自己）？
- **判断**：如果观众会觉得"你在说别人不是说我" → 致命伤。这类选题适合直播场景当案例讲，不适合做独立短视频。
- **原理**：高赞内容的共同点是观众被戳中（"有用的东西赚不了钱"31k——观众正在追求有用），而不是观众在旁观别人被批判。

**4.2 新鲜度检查：这个观点是不是已经被说烂了？**
- 如果用一句话总结这个选题的核心观点，普通人会觉得"这我早知道了"吗？
- **判断**：如果核心观点已经是常识级别的认知 → 致命伤。除非你有全新的角度或证据。
- **原理**：认知冲突的前提是"冲突"，如果观众已经认同这个观点，就没有冲突了。

**4.3 开头冲击力检查：能不能写出一句让人停下来的开头？**
- 尝试写 2-3 个开头第一句话。如果写不出一句让自己都觉得"这个有力"的开头 → 致命伤。
- **判断**：开头写不出来不是文案问题，是选题本身缺乏张力的信号。
- **原理**：好选题的开头是自然涌现的，不需要硬凑。如果要绞尽脑汁才能写出开头，说明选题的冲击力不够。

**致命伤判断**：
- 命中 1 条 → ✗ 直接否定，进入素材回收流程（见 Step 7）
- 0 条 → 继续评估

### Step 5: 标题张力检查

标题是否满足以下至少一项：
- [ ] 有对比/反差
- [ ] 有具体数字
- [ ] 有悬念/好奇心
- [ ] 有冲突/争议性
- [ ] 有时间承诺（借鉴 Dan Koe）
- [ ] 有竞争框架（借鉴 Dan Koe）

**悬念空间检查（2026年1月新增）：**
- 这个选题能否制造悬念？（提问而非给答案）
- 标题能否用"为什么"而不是"证明"？
- 选题是否有足够的认知冲突空间？
- 示例：
  - ❌ 无悬念空间："如何做XX"（直接给方法）
  - ✅ 有悬念空间："为什么XX"（引发好奇）
  - ❌ 无悬念空间："XX证明YY"（直接给结论）
  - ✅ 有悬念空间："XX，为什么YY"（留悬念）

**判断：**
- 勾选 2 个以上 → 标题有张力
- 勾选 1 个 → 标题一般
- 一个都没有 → 标题需要改造

### Step 6: 改造建议

如果选题有问题但没有致命伤，从以下角度改造：
1. **扩大话题**：从「做知识付费的人」扩展到「想赚钱的人」「想改变的人」
2. **加入数据**：把自己的真实数据融入选题
3. **制造冲突**：找到反直觉、反常规的角度
4. **加时间承诺**：借鉴 Dan Koe 的「6个月」「30分钟」框架
5. **加竞争框架**：借鉴 Dan Koe 的「超越99%」「领先他人」框架
6. **改标题**：增加对比、数字、悬念、冲突

### Step 7: 素材回收（v3.0 新增，仅否定选题时执行）

选题被否定不代表讨论过程中的洞察没用。执行以下回收：

**7.1 提取可复用金句/概念**
- 从讨论过程中提取有冲击力的表达（比喻、金句、概念框架）
- 这些素材可能在未来更强的选题里用到

**7.2 判断适合的使用场景**
- 适合做直播里的即兴案例？
- 适合做某条视频里的一个段落（而非整条视频）？
- 适合做推文/小红书笔记（文字比视频更适合这类观点）？

**7.3 记录到选题素材库**
- 将金句和使用建议记录到 `01-内容生产/内容素材库/` 对应文件
- 标注来源选题和否定原因，方便未来检索

**输出格式**：

## 第九部分：标题与封面规范（小红书）

### 9.1 标题规范（≤20字，含标点）
- 结果导向，少讲"方法"，多讲"状态/结果/承诺"
- 可用时间承诺（如：3个月、30天），但避免夸张
- 避免自我炫耀（如"越来越闲、越来越有钱"）
- 避免与事实不符（如"上班/不上班"叙事）
- 减少缩窄人群的词（如"自媒体/答疑"放正文，不放标题）
- 与封面文案尽量不重复，覆盖不同关键词以提升搜索
- **制造悬念，不给答案**（2026年1月新增）：用"为什么"而不是"证明"，用"问题"而不是"结论"

标题快速模板：
- 【时间承诺】+【状态结果】
  - 示例：3个月，我设计出一种不内耗的赚钱方式
- 【认知冲突】+【结果】
  - 示例：工作不累的人，都在做同一件事
- 【问题/痛点】+【可达成结果】
  - 示例：不内耗，也能持续赚钱
- 【提问式悬念】（2026年1月新增）
  - 示例：善良的李亚鹏，30年为什么赚不到钱

### 9.2 封面文案规范（3行，12–18字）
- 行数：优先3行；每行3–6字；第三行可稍长
- 断句：按意群断句，不拆词；每行包含一个信息点
- 逻辑：主题→方法/转折→结论/承诺
- 作用：承接标题，补足痛点/方法/承诺，避免与标题重复

三行模板：
```
[4字核心词]
[3–4字关键词]
[4–6字结论/承诺]
```
示例结构：
- 主题-方法-效果：AI起号 / 千万阅读 / 底层逻辑
- 现象-转折-结论：引流款 / 越便宜 / 生意越差
- 痛点-原因-结论：赚不到钱 / 是因为 / 每次都错过商机吗

### 9.3 标题×封面配合原则
- 标题负责"吸引点击与覆盖广人群"，封面负责"明确痛点与可达成结果"
- 标题与封面不重复表达，分头覆盖不同关键词
- 标题放"时间/结果/冲突"，封面放"痛点/方法/承诺"

### 9.4 质检清单（标题/封面）
- 标题≤20字（含标点）；封面总字数12–18字，单行3–6字
- 标题与封面是否互补、不重复？
- 是否避免"自运转/自动转/越来越闲/上班"等不当措辞？
- 是否避免缩窄词（自媒体/答疑）出现在标题？
- 是否包含时间承诺或明确结果？
