# dbs-video-workflow

**版本**：v1.0  
**创建日期**：2026-05-03  
**适用场景**：把一个想法/选题/产品，系统化地做成一条短视频的完整文案

---

## 核心逻辑

视频制作不是线性的"写稿→拍摄"，而是 **4 个心智阶段，每个阶段解决一个灵魂问题**：

```
阶段 1｜吃透选题
  灵魂问：你真的懂这个东西为什么存在吗？
  输出：选题的底层情绪 + 赛道定位 + 90% 痛点清单 + 数据支撑

阶段 2｜钩住人群
  灵魂问：3-90 岁的人凭什么点进来？
  输出：10+ 条爆款开头方案 + 痛点全覆盖清单

阶段 3｜承诺蜕变
  灵魂问：看完我这 5 分钟，用户从 A 变成什么 B？
  输出：A→B 路径（低能量 → 跃迁词 → 高能量 → 期许）+ 加过心理学/底层逻辑的 800 字

阶段 4｜聊透输出
  灵魂问：你能不拿稿子对着镜头说完吗？
  输出：内化后的大框架 + 删完废话的终稿
```

## 工作流程

### 入口

当用户说：
- 「帮我把这个选题做成视频」
- 「我想做一条关于 X 的视频」
- 「新建视频工作流 XXX」
- 「继续 XXX 项目」
- 使用 `/dbs-video-workflow` 或 `/视频工作流`

### 流程图

```
用户输入选题
    ↓
阶段 1：吃透选题 (phase1-understand.md)
  - 双层研究员预设（产品拆解 + 情绪穿透）
  - 可选：调用 /选题顾问 评估是否值得做
    ↓
  【Gate 1】给用户看双层知识卡，确认理解是否正确
    ↓
阶段 2：钩住人群 (phase2-hook.md)
  - 调用 /开头 skill 生成 10+ 条开头方案
  - 调用 /选题人群洞察 提炼人群痛点
    ↓
阶段 3：承诺蜕变 (phase3-transform.md)
  - 设计 A→B 蜕变路径
  - 加入心理学/底层逻辑（可调用 dbs-diagnosis）
    ↓
阶段 4：聊透输出 (phase4-output.md)
  - 内化成大框架
  - 调用 /逻辑延续 删废话
    ↓
输出：可直接对镜头说的视频文案
```

---

## Gate 机制

**阶段 1 → 阶段 2：强制 Gate**
- 理由：如果对选题的理解是错的，后面 7 步全在放大错误
- 形式：给用户看双层知识卡（产品拆解 + 情绪穿透），用户说"过"才继续

**其他阶段：自动流转**
- 理由：其他阶段的错最多是文案糙，改一下就行

## 状态管理

每个阶段结束后，自动保存 checkpoint 到：
```
~/.claude/projects/-Users-liujia/memory/video-workflow-state/
```

文件命名：
- `{timestamp}-phase1-knowledge-card.md`：阶段 1 的双层知识卡
- `{timestamp}-phase2-hooks.md`：阶段 2 的开头方案
- `{timestamp}-phase3-transform.md`：阶段 3 的 A→B 路径
- `{timestamp}-phase4-final.md`：阶段 4 的终稿

用户可以随时用 `/dbs-restore` 恢复之前的状态。

## 注意事项

1. **阶段 1 的 Gate 不能跳过**：如果理解错了，后面全废
2. **每个阶段的输出都会自动保存**：不用担心丢失
3. **可以随时中断**：用 `/dbs-save` 保存当前状态，下次用 `/dbs-restore` 继续
4. **不要跳阶段**：每个阶段的输出是下一个阶段的输入
5. **最终输出是"可以对镜头说的框架"**：不是逐字稿，是你完全理解后能脱稿说的大纲
