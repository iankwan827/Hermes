# Hermes 使用经验日志

## 2026-06-13

### 踩坑记录

- [Session: 配置切换搞错用户意图] 用户要求把主模型从 Token Plan 切到按量付费（sk key），agent 先说"文件没变"，然后才理解用户意图。改好后被飞书 agent 说"文件没变"误导，以为配置没生效。→ 解决方案：改配置后必须重启 Gateway（`hermes gateway stop && hermes gateway start`），不能只改文件。同时要主动读取当前配置确认方向是否正确，不要被其他 agent 的反馈带偏。

- [Session: 被记录欺骗不知道自己是谁] 用户问"你现在是 plan 的 mimo 还是 tokens 的 mimo"，agent 翻了历史记录还翻错了，先说 Token Plan，实际已经是按量付费。→ 解决方案：session header 就写着 Provider: xiaomi，配置文件也清清楚楚 sk key 在主位。应该直接看配置和 session header，不需要翻记录。下次遇到类似问题先读 `config.yaml` 确认当前状态。

- [Session: 新旧文件混淆] 用户从飞书下载了新的第三、四节课视频，但目录里已有旧版文件（2025年12月），agent 混淆了新旧文件，分析了旧文件还说"第一节课是旧的"被用户骂了。→ 解决方案：下载文件后先用 `ls -lt` 检查修改时间，区分新旧文件。分析前先确认目标文件是否是用户指定的那个。

- [抖音数据审核] V12（LOL老玩家）诊断文本中评论率记录为 0.16%，但实际正确值是 0.26%（5/1960）。原因是诊断使用了 24h 检查时的旧数据（3/1865=0.16%），但数据表已更新为最新数据（5/1960）。→ 解决方案：诊断结论中的数据引用必须与数据表当前值同步，不能用过期的中间状态数据。审核 cron job 的交叉检查成功发现了这个问题。

### 新发现

- [抖音数据分析] V13「西瓜打针」的标题悬念钩子（"两种人"模式）极其有效，2s跳出率12.23%是全号最低，播放量2381是全号最高。但互动指标（评论0、分享0、收藏0）全面归零，说明标题能吸引点击但内容无法激发互动。→ "标题钩子成功+互动归零"是需要关注的组合问题。

- [抖音数据趋势] 点赞率从V6的1.01%持续下降到V13的0.29%，连续6个视频在0.2-0.5%低位，是账号级系统性问题，不是个别视频问题。→ 需要专项分析内容情绪强度/共鸣点方面的原因。

- [抖音数据趋势] 2s跳出率从V10的19.52%上升到V12的25.36%，虽然V13大幅改善（12.23%），但V10-V12的上升趋势需要关注，可能是标题钩子效果减弱。

- [视频分析工具] video_analyze 工具可以分析本地视频文件，输出内容摘要、核心知识点、案例等。适合做课程视频的结构化整理。

- [抖音数据审核流程] 采集 cron + 审核 cron 的双 Agent 交叉检查流程验证有效，成功发现了数据不完整（V10/V11/V12缺失）、数据不同步（V12评论率）、汇总数据过期等问题。

### 用户偏好更新

- Agent 应该能自己判断当前使用的 provider/key，不需要每次都去翻记录或问用户。直接读配置文件和 session header 即可。
- 配置切换前要确认用户意图，不要搞反方向（用户说"切到 sk"是指主模型，不是备用）。
- 新下载的文件和旧文件要先区分时间戳再操作，不要混淆。
- 数据审核要严谨，发现数据不完整或过期要主动标记并修复。
- 视频文件分析前先确认文件是否是最新版本。
